## 4. 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| **搜索** | MySQL FULLTEXT INDEX | V1 数据量小，避免引入 ElasticSearch |
| **点赞去重** | Redis Set（IP hash + cookie id） | 高性能去重，定期同步计数到 MySQL |
| **敏感词** | 内存 Trie 树 + 定时从 DB 刷新 | 毫秒级检测，管理后台修改后实时同步 |
| **图片处理** | 前端压缩 + MinIO 存储 | 前端用 Canvas 压缩至 1200px，减小存储和带宽 |
| **限流** | Redis 滑动窗口 | 同 IP 1 分钟最多 3 帖、10 评论 |
| **会话** | Cookie（匿名 ID）+ JWT（管理后台） | C 端无需登录，B 端 JWT 认证 |
| **部署** | Docker Compose 一键启动 | 降低部署门槛，适合单服务器场景 |

## 5. 安全设计

| 防护项 | 方案 |
|--------|------|
| XSS 防御 | 输入转义（后端输出编码），CSP 头 |
| SQL 注入 | MyBatis-Plus 参数化查询 |
| CSRF | CORS 白名单 + SameSite Cookie |
| 暴力破解 | 登录失败 5 次锁定 15 分钟（Redis） |
| 恶意刷帖 | Redis 滑动窗口限流 |
| 图片安全 | 限制格式+大小，服务端二次校验 |
| 隐私保护 | IP 仅存哈希，不记录原始 IP |
